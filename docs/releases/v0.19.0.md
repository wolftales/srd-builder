## Overview

v0.19.0 implements **monsters v2.0.0 schema** with major structural improvements based on API analysis research (dnd5eapi.co, Open5e, 5e-bits). This release focuses on **data structure optimization** following proven patterns from established D&D APIs.

**Scope:** This release updates **monsters only**. Other datasets (equipment, spells, classes) will be updated in subsequent v0.19.x releases:
- v0.19.0 - Monsters v2.0.0 âœ…
- v0.19.1 - Equipment v2.0.0 (planned)
- v0.19.2 - Spells v2.0.0 (planned)
- v0.19.3 - Classes v2.0.0 (planned)

**Key Philosophy:** Hybrid flat+nested approach - complex fields use nested objects for structure and clarity.

## Breaking Changes

### âŒ Removed Fields
- **`summary`** - Removed as it was always a duplicate of the first trait's description with the trait name stripped (e.g., "Amphibious: The aboleth can breathe air and water" â†’ summary was just "The aboleth can breathe air and water"). No unique data was lost.

### ðŸ”„ Changed Fields

#### 0. Schema v2.0.0 Refinements (2025-12-23)

**Field Name Simplifications:**
Removed redundant prefixes from fields inside nested objects:

```json
// BEFORE: Redundant prefixes
{
  "damage": [{
    "damage_dice": "2d6+5",
    "damage_type": "bludgeoning",
    "damage_type_id": "bludgeoning"
  }],
  "range": {
    "range_normal": 150,
    "range_long": 600
  }
}

// AFTER: Cleaner field names
{
  "damage": [{
    "dice": "2d6+5",
    "type": "bludgeoning",
    "type_id": "bludgeoning"
  }],
  "range": {
    "normal": 150,
    "long": 600
  }
}
```

**hit_dice Cleanup:**
Changed from full formula to pure dice only (for game mechanics that reference hit dice count):
```json
// BEFORE
{
  "hit_dice": "7d10+21",  // Formula included modifiers
  "hit_points": {"average": 59, "formula": "7d10+21"}
}

// AFTER
{
  "hit_dice": "7d10",  // Pure dice only (XdY format)
  "hit_points": {"average": 59, "formula": "7d10+21"}  // Formula preserved here
}
```

**Ability Scores Nesting:**
Changed from flat structure to nested `{value, modifier}` objects:
```json
// BEFORE: Flat with separate modifier fields
{
  "ability_scores": {
    "strength": 21,
    "strength_modifier": 5,
    "dexterity": 9,
    "dexterity_modifier": -1
  }
}

// AFTER: Nested objects
{
  "ability_scores": {
    "strength": {
      "value": 21,
      "modifier": 5
    },
    "dexterity": {
      "value": 9,
      "modifier": -1
    }
  }
}
```

**Field Ordering:**
Properties reorganized to match D&D stat block conventions:
1. Identity (id, simple_name, name)
2. Combat Stats (armor_class, hit_points, speed)
3. Abilities (ability_scores)
4. Senses
5. Defenses (resistances, immunities, vulnerabilities)
6. Actions

#### 1. Armor Class Structure
**Before (v1.5.0):**
```json
{
  "armor_class": {
    "value": 17,
    "source": "natural armor"
  }
}
```

**After (v2.0.0):**
```json
{
  "armor_class": {
    "value": 17,
    "type": "natural armor",
    "type_id": "natural_armor"
  }
}
```

**Migration:**
- `source` â†’ `type` (same content, renamed)
- Added: `type_id` (normalized identifier)

---

#### 2. Speed Structure
**Before (v1.5.0):**
```json
{
  "speed": {
    "walk": 30,
    "swim": 60
  }
}
```

**After (v2.0.0):**
```json
{
  "speed": {
    "walk": 30,
    "swim": 60,
    "fly": 0,
    "burrow": 0,
    "climb": 0,
    "hover": false
  }
}
```

**Migration:**
- All movement types now explicit (0 for unused)
- `hover` is boolean flag (not nested value/condition)
- Removed: complex condition objects

---

#### 3. Senses Structure
**Before (v1.5.0):**
```json
{
  "senses": {
    "darkvision": 60,
    "passive_perception": 10
  }
}
```

**After (v2.0.0):**
```json
{
  "senses": {
    "darkvision": 60,
    "blindsight": 0,
    "tremorsense": 0,
    "truesight": 0,
    "passive_perception": 10
  }
}
```

**Migration:**
- All sense types now explicit (0 for unused)
- Guaranteed shape for queries

---

#### 4. Actions Structure
**Before (v1.5.0):**
```json
{
  "name": "Scimitar",
  "simple_name": "scimitar",
  "description": ["Melee Weapon Attack: +4 to hit, reach 5 ft., one target. Hit: 5 (1d6 + 2) slashing damage."]
}
```

**After (v2.0.0):**
```json
{
  "name": "Scimitar",
  "simple_name": "scimitar",
  "description": ["Melee Weapon Attack: +4 to hit, reach 5 ft., one target. Hit: 5 (1d6 + 2) slashing damage."],
  "attack_bonus": 4,
  "damage": [
    {
      "dice": "1d6+2",
      "type": "slashing",
      "type_id": "slashing"
    }
  ],
  "range": {
    "reach": 5
  }
}
```

**New Fields:**
- `attack_bonus` (integer): Extracted from "+X to hit"
- `damage` (array): Each element has `dice`, `type`, `type_id` (NOT `damage_dice`, `damage_type`, `damage_type_id`)
- `dc` (object): `{dc_value, dc_type, dc_type_id, success_type}` for saving throws
- `range` (object): `{reach?, normal?, long?}` for distance (NOT `range_normal`, `range_long`)

**Migration:**
- Description text preserved unchanged
- Structured fields added for queries
- Multi-damage attacks now arrays (e.g., bite + fire damage)
- Field names simplified (no redundant prefixes inside nested objects)

---

#### 5. Damage Resistances/Immunities/Vulnerabilities
**Before (v1.5.0):**
```json
{
  "damage_immunities": [
    {"type": "fire"}
  ]
}
```

**After (v2.0.0):**
```json
{
  "damage_immunities": [
    {
      "type": "fire",
      "type_id": "fire"
    }
  ]
}
```

**New Fields:**
- `type_id` (normalized identifier for cross-referencing)
- `conditions` (optional, e.g., "nonmagical")

---

#### 6. Condition Immunities
**Before (v1.5.0):**
```json
{
  "condition_immunities": [
    {"type": "charmed"}
  ]
}
```

**After (v2.0.0):**
```json
{
  "condition_immunities": [
    {
      "name": "charmed",
      "condition_id": "condition:charmed"
    }
  ]
}
```

**Migration:**
- `type` â†’ `name`
- Added: `condition_id` (cross-reference to condition dataset)

## New Features

### âœ¨ Structured Action Parsing
Actions, reactions, and legendary actions now include:
- Attack bonuses (extracted from "+X to hit")
- Structured damage arrays (multiple damage types supported)
- DC objects with ability, value, and success type
- Range objects (reach for melee, normal/long for ranged)

### ðŸ”— Cross-Reference IDs
All type-based fields now include normalized IDs:
- `type_id` for damage types and armor types
- `condition_id` for conditions (format: `condition:charmed`)
- `damage_type_id` in action damage arrays
- `dc_type_id` in saving throw objects

### ðŸ“Š Query Optimization
Guaranteed field presence:
- Speed: All movement types always present (0 if not applicable)
- Senses: All sense types always present (0 if not applicable)
- Simplifies queries - no need to check for field existence

## Migration Guide

### For API Consumers

**1. Update Ability Score References:**
```javascript
// Old (flat)
const strength = monster.ability_scores.strength;
const modifier = monster.ability_scores.strength_modifier;

// New (nested)
const strength = monster.ability_scores.strength.value;
const modifier = monster.ability_scores.strength.modifier;
```

**2. Update Armor Class References:**
```javascript
// Old
const armorType = monster.armor_class.source;

// New
const armorType = monster.armor_class.type;
const armorTypeId = monster.armor_class.type_id;
```

**3. Update Speed Queries:**
```javascript
// Old (field might not exist)
const swimSpeed = monster.speed.swim || 0;

// New (always exists)
const swimSpeed = monster.speed.swim;
const canHover = monster.speed.hover;
```

**4. Update Action Processing:**
```javascript
// Old
const attackBonus = parseActionText(action.description);

// New (structured with simplified field names)
const attackBonus = action.attack_bonus;
const firstDamage = action.damage?.[0];
const damageType = firstDamage?.type_id;  // NOT damage_type_id
const damageDice = firstDamage?.dice;     // NOT damage_dice
const rangeNormal = action.range?.normal; // NOT range_normal
```

**5. Update Condition References:**
```javascript
// Old
const immuneToCharmed = monster.condition_immunities.some(c => c.type === 'charmed');

// New
const immuneToCharmed = monster.condition_immunities.some(c => c.name === 'charmed');
const conditionRef = monster.condition_immunities[0].condition_id; // "condition:charmed"
```

**6. Update hit_dice Usage:**
```javascript
// Old (was formula with modifiers)
const hitDice = monster.hit_dice;  // "7d10+21"

// New (pure dice only)
const hitDice = monster.hit_dice;  // "7d10"
const hitPointFormula = monster.hit_points.formula;  // "7d10+21" (full formula here)
```

## Data Quality Improvements

### Validation Enhancements
- All 296 monsters validate against schema v2.0.0
- Stricter type requirements (no `additionalProperties: true` on nested objects)
- Pattern validation for IDs (`^[a-z0-9_]+$`)

### Parser Improvements
- Action parser detects attack bonuses, damage, DCs automatically
- Defense entries split conditional vs unconditional (e.g., "from nonmagical attacks")
- Hover detection in speed descriptions
- Success type detection for saving throws ("half", "none", "other")

## Performance Notes

### File Size
- Monsters dataset: ~1.2MB (increased ~15% due to explicit 0 values and structured fields)
- Tradeoff: Larger file size for faster queries (no null checks, no parsing)

### Query Performance
**Improved scenarios:**
- "Find all creatures with swimming speed" - no OR condition needed
- "Find all creatures with truesight" - direct numeric comparison
- "Find all fire-immune creatures" - use `type_id` for exact matches
