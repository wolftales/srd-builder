## What changed

### NEW `src/srd_builder/extract/datasets/extract_equipment_packs.py` (~285 lines)

The parser strategy:

1. **Pack-header regex** identifies each of the 7 sections:

   ```python
   _PACK_HEADER_RE = re.compile(
       r"(?P<name>[A-Z][a-z]+\u2019s Pack)\s*\((?P<cost>\d+)\s*gp\)\."
   )
   ```

2. **Section-end sentinel** stops the walk at the next section header:

   ```python
   _SECTION_END_RE = re.compile(r"\bTools\b\s+A tool helps")
   ```

3. **`Includes ŌĆ”` clause splitter** ŌĆö body text after the header is
   stripped of the `Includes ` prefix, the trailing
   "The pack also has 50 feet of hempen rope strapped to the side of it"
   sentence (Burglar's, Dungeoneer's, Explorer's) is detected and
   handled separately, then the prose is split on `, ` / ` and `
   boundaries.

4. **Phrase-to-item resolution table** maps the SRD's exact prose
   phrases to stable `item_id`s with quantity rules:

   ```python
   _PHRASE_TO_ITEM: dict[str, tuple[str, str, int | None]] = {
       "a backpack":       ("item:backpack",  "Backpack",   None),
       "5 candles":        ("item:candle",    "Candle",     None),  # leading digit ŌåÆ q=5
       "10 feet of string":("item:string_10_feet", "String, 10 feet", 1),  # N is part of identity
       "a bag of 1,000 ball bearings": ("item:ball_bearings_bag_of_1000",
                                        "Ball bearings, bag of 1,000", 1),
       ŌĆ”
   }
   ```

   The `qty_override` column distinguishes parsed-leading-digit phrases
   (`"5 candles"` ŌåÆ q=5) from N-bound identity phrases (`"10 feet of
   string"`, `"50 feet of hempen rope"`, `"2 blocks of incense"`,
   `"a bag of 1,000 ball bearings"`) where the digit is part of the
   item identity, not a quantity.

5. **Three-path token resolver** ŌĆö
   - article-prefixed: `"a backpack"` ŌåÆ looked up directly,
   - leading-digit: `"5 candles"` ŌåÆ strip digit, look up `"candles"`
     under the article-less form, set quantity from digit,
   - bare phrase: `"sealing wax"` / `"vestments"` ŌåÆ looked up as-is
     with q=1.

6. **Curly-apostrophe normalization** ŌĆö the SRD uses U+2019
   (`ŌĆÖ`); the assembler keys all use ASCII `'`. The extractor
   normalizes `BurglarŌĆÖs Pack` ŌåÆ `Burglar's Pack` at the output
   boundary so downstream code is unchanged.
