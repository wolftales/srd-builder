# Release v0.26.1 — Shared PDF Probe + Second Corruption Claim Disproved

**Release Date:** June 17, 2026
**Package Version:** 0.26.1
**Breaking Changes:** None
**Migration:** None

## Overview

v0.26.1 is a small, focused follow-up to v0.26.0 that turns the
provenance-reproducer pattern from a one-off into a primitive. It also
extends the v0.26.0 finding: a *second* "PDF text is corrupted" claim
has been mechanically disproved.

Two of the eight registered hand-curated data sources are now confirmed
retire-able in v0.27.0.

## What changed

### `utils/pdf_probe.py` — shared PDF text-probe primitive

A new ~70-line module that wraps the SRD-specific PDF read pattern:

- `open_pdf(path)` — context-managed `fitz.Document`
- `page_text(doc, idx)` — text from a single 0-based PDF index, with
  whitespace normalization on by default
- `pages_text(doc, indices)` — `{idx: text}` for a range
- `normalize_whitespace(text)` — collapses the SRD's
  `\t\r\xa0` (tab + CR + non-breaking space) word separators into single
  spaces
- `srd_page_to_pdf_index(srd_page)` — 1-based SRD page → 0-based fitz
  index

This is the primitive consumed by both the existing reproducer tests
*and* future real extractors (planned for v0.27.0). Lives under
`utils/` deliberately so it's not entangled with the
`extract/` ↔ `extraction/` directories that v0.26.2 will reorganize.

### Second reproducer test — `test_spell_lists_pages_class_headers_are_extractable`

Added to `tests/test_pdf_provenance.py`. Asserts that on pages 105–113
(`PAGE_INDEX["spell_lists"]`):

- All 8 class section headers appear (Bard Spells, Cleric Spells, Druid
  Spells, Paladin Spells, Ranger Spells, Sorcerer Spells, Warlock
  Spells, Wizard Spells)
- A sampling of well-known spells appear (Vicious Mockery, Sacred
  Flame, Druidcraft, Eldritch Blast, Fire Bolt)

Test passes. Both headers and body text are fully extractable after
whitespace normalization.

### Lineage reproducer refactored

The lineage reproducer (`test_lineage_pages_are_extractable_after_whitespace_normalization`)
now consumes `utils.pdf_probe` and `PAGE_INDEX["lineages"]` instead of
inline whitespace helpers and hardcoded page constants. Same behavior;
shared primitive.

### PROVENANCE registry updated

- `spell_class_targets.py` entry: `Reason` is now **DISPUTED** (was
  preliminary `pdf_corruption`). Reproducer linked. Retirement note
  for v0.27.0.
- New top-of-file banner: *"Two of the eight registered claims have
  already been disproved by mechanical probes. Treat any future 'PDF
  corrupted' rationale as a hypothesis until a reproducer test fails."*
- Status snapshot: 2 reproducer-backed entries (was 1), 6 awaiting.

### `AGENTS.md` — *PDF extraction discipline (no magic wands)*

A new section in `AGENTS.md` codifies the rule for future agents:

1. Before declaring any PDF region "corrupted," "unreadable," or
   "unextractable," add a reproducer test in
   `tests/test_pdf_provenance.py` using `utils.pdf_probe`.
2. If you cannot write a reproducer that fails on real PDF behavior,
   the region is not corrupted — it is just unparsed. Write the parser.
3. Hand-curated overrides require a `PROVENANCE.md` entry; any
   `pdf_corruption` claim must link a passing reproducer.
4. New page-text extraction must consume `utils.pdf_probe` and
   `utils.page_index`, not new `fitz.open()` calls or hardcoded page
   constants.

This rule exists because the prior path — declare corrupted,
transcribe by hand, move on — quietly accumulated ~2,000 lines of
hand-curated data across the codebase, most of which now appears to
be needless.

### BACKLOG ticket for v0.26.2 structural cleanup

The audit done in service of this release surfaced organizational debt
that's *not* in scope for v0.26.1 but should not be lost. A new
BACKLOG section captures:

1. Naming inconsistency — `extract/` (per-dataset bespoke modules)
   vs. `extraction/` (the table engine). Same pipeline layer, different
   shapes. Proposed v0.26.2 cutover plan included.
2. Misplaced data and config — hand-curated data and engine config
   live in five different homes (`postprocess/`, `extraction/`,
   `scripts/`, `utils/`, `srd_5_1/`, `data/`, `assemble/`) with no
   organizing principle. Proposed target: per-ruleset `data/` and
   `config/` dirs under `rulesets/srd_5_1/` alongside the existing
   PDF and `raw/` dirs.
3. **Historical framing** — this is the third attempt at
   standardization. Attempts 1 and 2 left artifacts (the `extraction/`
   table engine and `postprocess/engine.py` are attempt #2's surviving
   work; attempt #1 is gone except for some schema 2.0.0 fingerprints).
   v0.26.2 + v0.27.0 are explicitly framed as **consolidating attempt
   #2**, not designing from scratch.

## What's next (v0.26.2 → v0.27.0)

- **v0.26.2** (structural cleanup): rename `extraction/` → `extract/`,
  move bespoke `extract_*.py` modules to a legacy subdir, move
  hand-curated ruleset data into `rulesets/srd_5_1/data/`,
  consolidate engine config into `rulesets/srd_5_1/config/`. One
  commit per logical move; no behavior change.
- **v0.27.0**: write real `extract_lineages.py` and
  `extract_spell_classes.py` on top of `utils.pdf_probe` +
  `PAGE_INDEX`. Retire `LINEAGE_DATA` (`lineage_targets.py`) and
  `SPELL_CLASSES` (`spell_class_targets.py`) — together about
  ~1,400 lines of hand-curated data. Probe `class_targets.py`
  features list with the same pattern; if its corruption-rationale
  also fails to reproduce, retire that too.

Goal: bring the PROVENANCE registry from 8 entries down to 4–5,
displacing the corruption-rationale entries with real extraction
rather than transcription.
