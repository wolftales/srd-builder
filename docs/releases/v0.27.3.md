## What changed

### Damage-formula regex extended to all three SRD phrasings

`parse_poison_descriptions._parse_single_description` previously
matched only `takes?\s+...\s+damage`. The SRD uses three phrasings:

| Phrasing | Poisons |
| --- | --- |
| `"takes X (NdM) poison damage"` | `assassins_blood` |
| `"take X (NdM) poison damage"` | `burnt_othur_fumes`, `pale_tincture` |
| `"taking X (NdM) poison damage on a failed save"` | `midnight_tears`, `purple_worm_poison`, `serpent_venom`, `wyvern_poison` |

A single regex now covers all three:

```python
re.search(
    r"tak(?:es?|ing)\s+(\d+)\s*\(([^)]+)\)\s+(\w+)\s+damage",
    text,
    re.IGNORECASE,
)
```

The four delayed-damage poisons that v0.27.1 flagged as the remaining
blocker now parse cleanly.

### `type_id` field emitted on damage block

The legacy `POISON_DESCRIPTIONS` shape included
`"type_id": "damage:poison"` on every damage block. The extractor now
emits that field too, derived from the matched damage type:

```python
result["damage"] = {
    "average": int(damage_match.group(1)),
    "formula": damage_match.group(2),
    "type": damage_type,
    "type_id": f"damage:{damage_type}",
}
```

### Leading section header stripped from description body

The live extractor used to keep the leading `"Name (Type). "` section
header (e.g. `"Assassin's Blood (Ingested). A creature subjected to
this poison must…"`) at the start of the description string; the
hand-curated dict did not. A targeted strip aligns the two:

```python
header_match = re.match(
    rf"\s*{re.escape(name)}\s*\([A-Za-z]+\)\.\s*",
    text,
)
if header_match:
    text = text[header_match.end() :]
```

### Cutover: `parse_poisons_table.py`

* Removed `from ..rulesets.srd_5_1.poison_descriptions import POISON_DESCRIPTIONS`.
* Removed the "prefer manual, fall back to extracted" branch.
* The merge now consumes `descriptions_map` (the live extractor's
  output) directly. The merge surface and field names
  (`description` / `save` / `damage`) are unchanged.

### Fixture regenerated

[tests/fixtures/srd_5_1/raw/poisons.json](../../tests/fixtures/srd_5_1/raw/poisons.json)
was incomplete: it omitted `assassins_blood` from the
`poison_descriptions` block on the assumption that the legacy
`POISON_DESCRIPTIONS` dict would fill it in at merge time. The fixture
was regenerated from live extractor output so the golden test is
self-contained.
