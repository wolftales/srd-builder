# v0.27.7 — `equipment_extended.py` provenance hardening (BACKLOG proposal 2)

## TL;DR

The 9 hand-curated items in `EXTENDED_EQUIPMENT` — the *last* live
entry in the PROVENANCE registry after the v0.27.x retirement line —
now carry a typed `_provenance` block instead of a free-text `_note`
field. The previous `_note` field was silently violating
`equipment.schema.json`'s `additionalProperties: false` (caught only
because per-item validation wasn't running on the assemble-only path);
downstream consumers can now programmatically filter inferred records
(`item.get("_provenance", {}).get("source") == "inferred"`) instead of
grepping prose.

- **`ExtendedProvenance` TypedDict** added to
  `src/srd_builder/assemble/equipment_extended.py` (`source`, `reason`,
  `referenced_by`, `module`, `estimates`).
- **`equipment.schema.json` 2.1.0 → 2.2.0** — new `_provenance` property
  with full required-fields enum (`source: "inferred"`, `reason:
  "pack_cross_reference" | "monster_cross_reference"`, `estimates:
  array<"cost" | "weight">`, etc.) and `additionalProperties: false`.
- **+5 new tests** in `tests/test_equipment_extended_provenance.py`
  pinning shape, the 8-vs-1 pack-vs-monster split, and round-trip
  through `get_extended_equipment()`.
- Closes the **last open follow-up** in
  [docs/PROVENANCE.md](../PROVENANCE.md) and BACKLOG proposal 2.
- **`563 passed`** (5 new, 0 new failures; 17 pre-existing
  fixture-version-drift failures unchanged).

## Background

The v0.27.x retirement line (v0.27.0 → v0.27.6) deleted ~2,856 lines
across 6 hand-curated modules by writing live PDF extractors and
disproving the original "PDF text is corrupted" / "PDF missing"
rationales. After v0.27.6 the PROVENANCE registry had exactly one LIVE
entry remaining: `equipment_extended.py` — a `cross_reference_glue`
module whose records do not appear in the SRD PDF at all (they are
author-invented stubs for items referenced by equipment packs and
monster stat blocks but never indexed in the equipment tables).

This module is a *legitimate* hand-curated entry — there is no PDF to
extract from, so it cannot be retired the same way the other six were.
But each record carried free-text justification in a `_note: str`
field, which BACKLOG proposal 2 had flagged as a candidate for
promotion to a structured `_provenance` block so consumers could
filter or flag inferred records in UIs.

This release closes that proposal.

## What changed

### `src/srd_builder/assemble/equipment_extended.py` (rewritten, ~210 lines)

**Was:** 167 lines with `_note: str` on each of 9 items, including
one item with a doubled `"Cost/weight estimated. Cost/weight
estimated."` string that nobody had caught.

**Now:**

```python
class ExtendedProvenance(TypedDict):
    source: Literal["inferred"]
    reason: Literal["pack_cross_reference", "monster_cross_reference"]
    referenced_by: str
    module: str
    estimates: list[Literal["cost", "weight"]]

_MODULE = "equipment_extended.py"

def _pack_provenance(pack_name: str) -> ExtendedProvenance:
    return {
        "source": "inferred",
        "reason": "pack_cross_reference",
        "referenced_by": pack_name,
        "module": _MODULE,
        "estimates": ["cost", "weight"],
    }

def _monster_provenance(referenced_by: str) -> ExtendedProvenance:
    return {
        "source": "inferred",
        "reason": "monster_cross_reference",
        "referenced_by": referenced_by,
        "module": _MODULE,
        "estimates": [],
    }
```

**Coverage of all 9 items:**

| Item ID | Provenance helper | `referenced_by` |
| --- | --- | --- |
| `item:string_10_feet` | `_pack_provenance` | `"Burglar's Pack"` |
| `item:alms_box` | `_pack_provenance` | `"Priest's Pack"` |
| `item:incense_2_blocks` | `_pack_provenance` | `"Priest's Pack"` |
| `item:censer` | `_pack_provenance` | `"Priest's Pack"` |
| `item:vestments` | `_pack_provenance` | `"Priest's Pack"` |
| `item:book_of_lore` | `_pack_provenance` | `"Scholar's Pack"` |
| `item:bag_of_sand_little` | `_pack_provenance` | `"Scholar's Pack"` |
| `item:knife_small` | `_pack_provenance` | `"Scholar's Pack"` |
| `item:natural_armor` | `_monster_provenance` | `"monster_stat_blocks"` |

`natural_armor` is the only non-pack record (it backs monster stat
blocks where AC includes a "Natural Armor" tag); its `cost` is `0`
rather than an estimated buy-price, hence `estimates: []`.

`get_extended_equipment(source: str)` is unchanged.

### `schemas/equipment.schema.json` 2.1.0 → 2.2.0

New `_provenance` property added to the item object schema (still
`additionalProperties: false`):

```json
"_provenance": {
  "type": "object",
  "description": "Structured provenance for non-PDF-extracted records...",
  "required": ["source", "reason", "referenced_by", "module", "estimates"],
  "additionalProperties": false,
  "properties": {
    "source":        {"type": "string", "enum": ["inferred"]},
    "reason":        {"type": "string",
                      "enum": ["pack_cross_reference",
                               "monster_cross_reference"]},
    "referenced_by": {"type": "string"},
    "module":        {"type": "string"},
    "estimates":     {"type": "array",
                      "items": {"type": "string",
                                "enum": ["cost", "weight"]}}
  }
}
```

`last_modified` bumped to `2026-06-18`. Fixture
`tests/fixtures/srd_5_1/normalized/equipment.json` `schema_version`
bumped 2.1.0 → 2.2.0 to match.

### NEW `tests/test_equipment_extended_provenance.py` (~70 lines, 5 tests)

Five focused assertions on the in-memory `EXTENDED_EQUIPMENT` list and
on the `get_extended_equipment()` factory output:

1. **`test_every_extended_item_has_provenance_block`** — every record
   has `_provenance`, and no record retains the old `_note`.
2. **`test_provenance_block_shape_is_uniform`** — keys, `source`
   value, `reason` in valid set, non-empty `referenced_by`, exact
   `module` match, `estimates` is a subset of `{"cost", "weight"}`.
3. **`test_pack_referenced_items_estimate_cost_and_weight`** — exactly
   8 pack items, all with `estimates == ["cost", "weight"]` and
   `referenced_by` ending in `"'s Pack"`.
4. **`test_monster_referenced_natural_armor_estimates_nothing`** —
   exactly 1 monster record (`item:natural_armor`),
   `estimates == []`, `referenced_by == "monster_stat_blocks"`.
5. **`test_get_extended_equipment_preserves_provenance`** — round-trip
   via the factory preserves `_provenance` and applies the wrapping
   `"... (extended)"` source tag.
